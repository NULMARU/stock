# 스코어링 방법론 (METHODOLOGY)

> 대응 버전: `scoring_config.py`의 `METHODOLOGY_VERSION`
> 규칙: 임계값·가중치 조정은 **config 파일만** 수정하고, 이 문서에 변경 이력을 남긴다.

## 1. 개요

초보 투자자 학습용 5축 체크리스트 점수. 투자 조언이 아니라 "지표를 어떻게 읽는지"를
보여주는 것이 목적이므로, 모든 판정은 `check = {label, pass, detail}` 형태로
JSON에 그대로 남겨 프론트에서 "왜 이 점수인가"를 펼쳐 볼 수 있게 한다 (spec §4.1 투명성).

- 5축: 밸류에이션 / 성장성 / 수익성 / 재무건전성 / 모멘텀
- 축당 0~5점, 총점 0~25점. 각 check 통과 시 1점 (통과 수 = 점수).
- 가중치: 5축 동일(1.0). 단순성 우선 (spec §4.6).

## 2. 축별 판정 기준

### 2.1 밸류에이션 (valuation)

**흑자 기업** — 5개 check:

| check | 임계값 | 근거 |
|---|---|---|
| PER ≤ 15배 | `PE_CHEAP` | 전통적 저평가 기준 (그레이엄 계열) |
| PER ≤ 30배 | `PE_FAIR` | 성장주 포함 완화 기준 |
| PER ≤ 그룹 중앙값 | — | 상대 비교 병용 (spec §4.2). 그룹 = market + theme[0] |
| PBR ≤ 3배 | `PB_FAIR` | 무형가치 프리미엄 허용 상한 |
| PSR ≤ 5배 | `PS_FAIR` | 매출 대비 과열 보조 지표 |

**과열 캡 (spec §4.4)**: PER > 200배(`PE_OVERHEAT`)이면 세부 체크를 생략하고
"과열" 단일 판정으로 점수를 1점으로 캡한다. 캠브리콘(PER 수천 배) 같은 극단 사례에서
체크가 무의미해지는 것을 방지.

**적자 기업 (PER null/음수)** — PSR + 매출성장률 대체 판정 (spec §4.3):

| check | 임계값 |
|---|---|
| PSR ≤ 3배 | `PS_CHEAP` |
| PSR ≤ 10배 | `PS_FAIR_LOSS` |
| 매출성장률 ≥ 20% | `REV_GROWTH_STRONG` |
| 매출성장률 ≥ 40% | `REV_GROWTH_VERY_STRONG` |
| forward PER 존재 | 흑자 전환 기대 (애널리스트 추정) |

적자 기업에는 "적자 기업" riskFlag를 부여한다.

### 2.2 성장성 (growth)

| check | 임계값 |
|---|---|
| 매출성장률 ≥ 10% | `REV_GROWTH_OK` |
| 매출성장률 ≥ 25% | `REV_GROWTH_HIGH` |
| 순이익성장률 ≥ 10% | `EARN_GROWTH_OK` |
| 순이익성장률 ≥ 25% | `EARN_GROWTH_HIGH` |
| 이익 개선 기대 | 흑자: forwardPE ≤ trailingPE / 적자: forwardPE 존재 |

지표가 없으면 해당 check는 불통과 + detail에 "데이터 없음"으로 기록한다.

### 2.3 수익성 (profitability)

**흑자 기업**: ROE ≥ 8% / ≥ 15%, 영업이익률 ≥ 10% / ≥ 20%, 순이익률 ≥ 10%.
ROE 15%는 S&P 500 평균(15~20%) 하단을 기준점으로 사용 (glossary ROE 항목과 일치).

**적자 기업** (spec §4.3: 영업이익률 추세로 대체):

| check | 임계값 |
|---|---|
| 매출총이익률 ≥ 30% | `GM_OK` |
| 매출총이익률 ≥ 50% | `GM_GOOD` |
| 영업손실 축소 추세 | 최근 4분기 영업이익이 첫 분기 대비 개선 |
| 최근 분기 영업 흑자 | 가장 최근 분기 영업이익 > 0 |
| forward PER 존재 | 흑자 전환 기대 |

분기 추세는 yfinance `quarterly_income_stmt`의 Operating Income을 사용하며,
적자 기업일 때만 추가 조회해 API 부하를 줄인다.

### 2.4 재무건전성 (financialHealth)

| check | 임계값 | 근거 |
|---|---|---|
| 부채비율 ≤ 100% | `DE_OK` | 일반적 안정 기준 (glossary 부채비율 항목) |
| 부채비율 ≤ 50% | `DE_GOOD` | 보수적 재무 구조 |
| 유동비율 ≥ 1.5 | `CURRENT_RATIO_OK` | 단기 지급능력 |
| 총현금 > 총부채 | — | 실질 무차입 |
| 잉여현금흐름 > 0 | — | 현금 창출력 |

### 2.5 모멘텀 (momentum, spec §4.5)

1년 일봉에서 계산:

- 52주 위치 = (현재가 − 52주 최저) ÷ (52주 최고 − 52주 최저)
- 3개월 수익률 = 현재가 ÷ 63거래일 전 종가 − 1

| check | 임계값 |
|---|---|
| 52주 위치 ≥ 40% | `POS_OK` |
| 52주 위치 ≥ 60% | `POS_GOOD` |
| 3개월 수익률 > 0% | `RET3M_OK` |
| 3개월 수익률 ≥ 10% | `RET3M_GOOD` |
| 현재가 > 50일 이동평균 | `MA_WINDOW` |

**과열 감점**: 52주 위치 ≥ 95%(`OVERHEAT_POS`)이면 "52주 고점 근처 과열" check를
불통과로 기록하고 1점 감점한다 (최소 0점).

## 3. 비교 컨텍스트 (sectorAvg)

- 그룹: `market + theme[0]` (예: "US:AI플랫폼").
- 그룹 내 양수 PER/PSR의 **중앙값**을 pe/ps로 저장. 극단치(캠브리콘 등)에
  평균이 오염되지 않도록 평균 대신 중앙값 사용.
- 그룹 구성원이 1개뿐이면 자기 자신이 중앙값이 된다(비교 의미 없음) — UI에서
  "비교 그룹 데이터 부족"으로 안내 가능.

## 4. 리스크 플래그 (riskFlags)

자동 판정:

- `적자 기업` — PER null/음수이고 순이익률도 0 이하
- `고PER (100배 이상)` — PER ≥ 100
- `고변동성` — 베타 ≥ 2.0
- `중국 A주 조회 전용` — 티커가 `.SS`/`.SZ` (후강퉁/선강퉁 요건, 실투자 제한)
- `소형주` — 시총이 통화별 기준 미만 (USD $20억 / KRW 3조 / HKD·CNY 150억)

수동 플래그: `universe.py`의 `extraRiskFlags` (예: 신규 상장, 극단적 밸류에이션).

## 5. insights (자동 한 줄 해석)

- 종목당 3~5개 한국어 문장. 토스식 해요체 구어체.
- 생성 순서: 밸류에이션 → 성장 → 수익성 → 재무 → 모멘텀 → 베타. 지표가 null이면
  그 지표 문장은 생략하고, 5개를 초과하면 앞에서부터 5개만 유지.
- 숫자는 항상 해석을 동반한다 ("업종 평균보다 높아요" 등, glossary 시사점 4 반영).

## 6. 데이터 수집·품질 규칙

- 소스: yfinance `Ticker.info` + `history(period="1y")`.
- priceHistory는 최근 1년 일봉을 주 1개(약 52포인트)로 샘플링해 JSON 크기 절감.
- 티커당 재시도 1회, 종목 간 0.4초 간격. 실패 시 건너뛰고 경고 출력.
- 최소 성공 22개 미만이면 파이프라인 실패(exit 1)로 간주.
- 원시 응답은 `scripts/.cache/`에 당일 캐시해 재실행·부분 실패 복구에 사용
  (`--refresh`로 무시 가능).
- `asOf`는 파이프라인 실행일(KST). UI에 기준일 표기 필수.
