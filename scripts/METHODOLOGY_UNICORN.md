# 유니콘(3축 가치주) 평가 알고리즘 방법론

> 기준 문서: `research/unicorn_research.md` (가치주조사_E1, 2026-07-21)
> 구현: 유니콘파이프라인_C1 · 방법론 버전은 `scripts/unicorn_config.py`의 `UNICORN_METHODOLOGY_VERSION`
> 출력: `src/data/unicorns.json` + `public/data/unicorns.json` (동일 내용, `UnicornData` 스키마)

## 1. 개요

저평가(밸류에이션 40점) + 내실(퀄리티 35점) + 성장(25점) 3축 100점 만점으로
후보 풀 60종(`scripts/unicorn_candidates.py`, 리서치 §3)을 채점하고,
총점 65점 이상만 유니콘 탭에 노출한다.

- 등급: 80+ = A, 65~79 = B, 65 미만 = C(탭 미노출). (리서치의 S/A/B/C를 프론트 타입 `UnicornGrade = "A"|"B"|"C"`에 맞춰 S→A로 매핑)
- T3(과잉 레버리지) 발동 시 최고 등급 B로 캡.
- 모든 판정은 `{label, pass, detail}` 체크로 기록되며 총점은 항상 세 축 점수의 합과 일치한다.
- 결측(null) 항목은 배점에서 제외하고 `(획득/응시만점)×축 가중치`로 정규화 (리서치 §2 원칙 ③).
- `asOf`·`methodologyVersion`을 결과에 기록 (리서치 §2 보정 규칙 4번).

## 2. 채점표와 임계값 근거

모든 숫자는 `unicorn_config.py`에 있다. 근거는 리서치 §2 표 A/B/C와 §1 방법론 비교:

| 축 | 지표 | 만점/부분/0점 | 배점 | 근거 (리서치 인용) |
|---|---|---|---|---|
| A1 | trailingPE | ≤15 / ~22 / >35 | 10 | 그레이엄 방어적 투자자 PER 15, 버핏 스크린 PER<25 |
| A2 | forwardPE | ≤12 / ~18 / >25 | 8 | 마법공식 EY 10% ≈ PER 10 |
| A3 | priceToBook | ≤1.5 / ~3.0 / >5 | 6 | 그레이엄 PBR 1.5 (PER×PBR ≤ 22.5의 절반) |
| A4 | PEG | ≤1.0 / ~1.5 / >2.0 또는 성장률≤0 | 8 | 린치 PEG 1.0 = 적정 |
| A5 | FCF 수익률 | ≥8% / 4~8% / <2%·FCF적자 | 8 | 마법공식 EY·버핏 이익수익률 근사 (ValueSense Shell 사례) |
| B1 | ROE | ≥15% / 8~15% / <0 | 10 | 버핏 15%, AQR QMJ 수익성 |
| B2 | ROA | ≥7% / 4~7% / <0 | 5 | Piotroski 수익성, 마법공식 ROC 대용 |
| B3 | 영업이익률 | ≥15% / 8~15% / <0 | 7 | Novy-Marx 수익성 계열 |
| B4 | 순이익 흑자 AND FCF 흑자 | 충족/미충족 | 3 | Piotroski ①②, 이익의 질 |
| B5 | debtToEquity(% 단위) | ≤50 / ~100 / >150 | 6 | 버핏 D/E<0.5, QMJ 안전성 |
| B6 | currentRatio | ≥1.5 / 1.0~1.5 / <1.0 | 4 | Piotroski 유동성 |
| C1 | 매출 성장률 | ≥10% / 3~10% / <0 | 7 | 버핏 스크린 성장 10% 계열 |
| C2 | 이익 성장률 | ≥10% / 5~10% / <0 | 8 | QMJ 성장 기둥 (부분 기준은 §5 v1.1 참조) |
| C3 | forwardPE < trailingPE | 충족/미충족 | 5 | PEG 사전 필터 |
| C4 | 배당수익률 | ≥3% / 1~3% / 0% | 5 | QMJ 주주환원, PEGY 사상 |

### 금융 섹터 보정 (리서치 §2 '시장·섹터 보정')

은행·보험·지주는 예금·보험부채가 부채로 잡혀 B5·B6이 왜곡되므로(마법공식의 "금융 제외" 규칙에 준함):
- B5·B6(10점) 제외 → 동일 배점의 `PBR ≤ 1.0 가산` 체크로 대체
- B1 ROE 만점 기준 15% → 10% 완화 (부분 5%)
- [v1.1] B2 ROA 기준 7% → 1% 완화 (부분 0.5%) — 은행·보험은 레버리지 모델상 ROA ~1%가 우량
- 판정: yfinance `sector == "Financial Services"` 우선, 결측 시 후보 풀의 financial 힌트

### 가치 함정 필터 (리서치 §2 T1~T4)

| 규칙 | 조건 | 처리 |
|---|---|---|
| T1 구조적 쇠퇴 | 매출성장 <0 AND 이익성장 <−20% | 성장 축 −15 + 플래그 |
| T2 저PBR 함정 | PBR <0.5 AND ROE <5% | A3 무효화 + 플래그 |
| T2f 금융 저ROE 함정 [v1.1] | 금융 AND ROE <8% | B1 무효화 + 플래그 |
| T3 과잉 레버리지 | D/E >200 AND 유동비율 <1.0 | 퀄리티 축 −10 + 등급 B 캡 + 플래그 |
| T4 배당 함정 | 배당 ≥6% AND (FCF 적자 또는 순이익 적자) | C4 0점 + 플래그 |

## 3. 데이터 처리 규칙

- 수집: yfinance `info`만 사용, 티커당 재시도 1회, 당일 캐시(`scripts/.cache/unicorn_*`) 재사용, `--refresh`로 무시 가능.
- 미국 클래스주는 대시 형식으로 변환 (`BRK.B → BRK-B`; yfinance 1.5.1이 점 형식 미지원).
- PER·PBR ≤ 0은 이상치로 간주해 결측 처리 (적자 음수 PER, BRK의 PBR 0.001 같은 결산 공백 값).
- PEG: `pegRatio` 우선, 결측 또는 이상치(≤0 또는 >5)이면 `forwardPE / (earningsGrowth×100)` 재계산 — 이때 성장률은 50% 상한(일회성 이익 급증 왜곡 방지). 성장률 ≤0이면 PEG 0점(결측 제외 아님).
- 배당수익률 결측(None)은 묰배당(0%)으로 간주해 0점 채점 (리서치 C4 0점 기준이 '0%'이므로).
- 평가 제외: 지표 결측 50% 초과(리서치 §5-4-a), 시총 하한 미달(미국 $2B·한국 5천억·중국 HK$200억, §5-4-b), 수집 실패.
- 한국 금융주 주의: yfinance가 105560.KS·139130.KS 등의 trailingPE·priceToBook을 현재 제공하지 않음(2026-07-21 확인). 결측 정규화로 평가는 진행되며, 이것이 T2f를 ROE 단독 판정으로 설계한 이유.

## 5. 변경 이력 (자기개선: 임계값은 config만 바꾸고 여기 기록)

### v1.1.0 (2026-07-21) — 회귀 테스트 캘리브레이션

1. **PEG 이상치 재계산** (`PEG_OUTLIER=5.0`, `PEG_GROWTH_CAP=0.50`): BRK.B의 yfinance pegRatio가 10.06(5년 예상 성장률 기반 왜곡)으로 반환되는 것을 확인. 이상치 시 리서치 §1-3의 폴스백(forwardPE/성장률)으로 재계산하되, BRK의 일회성 이익 급증(+119.6%)이 그대로 반영되지 않도록 성장률 50% 상한.
2. **금융 ROA 완화** (`FIN_ROA_FULL=0.01`, `FIN_ROA_PARTIAL=0.005`): 리서치 B2(7%)는 비금융 기준으로, 은행 ROA(KB 0.76%, DGB 0.45%)에 적용하면 전원 0점이 되어 변별력 상실. 은행권 ROA 1% = 우량이라는 업계 통념에 근거.
3. **T2f 금융 저ROE 함정 신설** (`T2F_ROE=0.08`): 리서치 §4가 DGB를 'T2 발동 대상'으로 지목했으나 한국 금융주는 yfinance PBR 결측으로 원형 T2가 작동 불가. 함정의 본질(구조적 저ROE의 영구 할인)을 ROE<8% 단독으로 판정. DGB(7.26%) 발동·KB(9.99%) 미발동 확인.
4. **C2 부분 기준 3%→5%** (`EARG_PARTIAL=0.05`): 3~5% 명목 이익 성장은 물가상승률 수준으로 실질 성장으로 보기 어려움. DGB(3.0%)가 부분 점수를 받아 탈락 기준(≤50)을 웃도는 문제(51.1점)를 원칙 있는 조정으로 해소.
5. **PER·PBR ≤0 결측 처리**: BRK.B의 PBR 0.001 같은 이상치가 만점으로 계산되던 데이터 위생 문제 수정.

## 6. 실행 방법

```bash
python3 scripts/unicorn_evaluate.py          # 60종 평가 → unicorns.json 2곳 저장
python3 scripts/unicorn_evaluate.py --refresh # 캐시 무시 재수집
python3 scripts/test_unicorn.py              # 단위+회귀 테스트
python3 scripts/auto_unicorn.py              # 평가 + git 커밋/푸시
python3 scripts/auto_unicorn.py --no-git     # 평가만
```

최근 실행(2026-07-21): 평가 58종(제외 2종: 시총 하한 미달), 통과 25종(43%).
통과 비율이 리서치 목표 구간(15~40%)을 소폭 상회하는데, 이는 후보 풀이 애초에
가치주로 큐레이션된 모집단이라는 점을 감안해야 한다(리서치 §5-5: 후보는 통과 보장 아님).
탭 노출 결정은 갱신 시점 점수로만 이뤄지므로, 주간 갱신 시 비율은 변동한다.
